# Floor versions — the LOWEST version of every direct dep we advertise
# in pyproject.toml. Used by the install-smoke-min CI job to verify our
# advertised ranges aren't lies.
#
# When you bump a `>=` lower bound in pyproject.toml, bump the matching
# pin here. When pip resolves the install with `--constraint floor.txt`,
# pip MUST pick this exact version (constraints "==" beat range lower
# bounds). The install-smoke-min job then runs the 5-liner against this
# floor, proving the advertised range actually works.
#
# Single source of truth — keep in lockstep with pyproject.toml's
# `dependencies` and the relevant extras.
#
# Not for end users. End users install arbez normally and let pip pick
# the latest of everything within our advertised ranges.
#
# DEPENDABOT-IGNORED (S-085, 2026-05-18): every dep pinned in this file
# is listed in `.github/dependabot.yml` under the `pip` ecosystem's
# `ignore:` block so dependabot never proposes a floor bump
# automatically. Floor bumps must be DELIBERATE (a paired pyproject +
# floor.txt change in a real ADR — e.g. "we're dropping py3.10 support,
# so the numpy floor moves from 1.24 to X"). The original false-fire
# was an earlier dependabot misfire (2026-05-18); see ADR S-085 in DECISIONS.md. When
# adding a new dep here, ALSO add it to dependabot.yml's `ignore:`
# block — otherwise the same misfire repeats on the next patch cycle.
# Note: security alerts (dependabot's separate vulnerability flow) are
# NOT affected by `ignore:` and will still open PRs for known-vulnerable
# versions of these deps.
